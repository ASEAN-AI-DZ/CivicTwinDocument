рҹ”‘ Core ServicesвҖӢ

1. Digital Twin Service вӯҗвҖӢ

Purpose: Real-time city modeling

  • рҹҢҚ Build a city network graph
  • рҹ“Қ Integrate geospatial data (PostGIS) for roads, intersections, and infrastructure
  • вҸұпёҸ Continuous updates from sensors, cameras, IoT
  • рҹ’ҫ Store state + history

Technologies: Node.js, PostgreSQL + PostGIS, Redis, Kafka
API Port: 3001
Documentation: DigitalTwinService/Readme.md

# Health check
curl http://localhost:3001/api/digital-twin/health

# Get zones
curl http://localhost:3001/api/digital-twin/zones

2. AI Prediction Service рҹӨ–вҖӢ

Purpose: Forecast & alert on incidents

  • рҹҡ— Traffic Prediction: Traffic flow forecasting (15вҖ“60 minutes ahead)
  • рҹ’§ Flooding Alert: Flood warnings based on weather + sensors
  • рҹ“Ҡ Cascade Effects: Simulate cascade impacts
  • рҹ’° Socioeconomic Impact: Predict economic-social impacts (with Amazon Nova)

Technologies: Python, FastAPI, TensorFlow/PyTorch
API Port: 8002
Documentation: AIMLService/Readme.md

# Get traffic prediction
curl http://localhost:8002/api/predictions/traffic/road_123?horizonMinutes=60

# Get flooding alert
curl http://localhost:8002/api/predictions/flooding/city

3. Simulation & What-If Service рҹҺ®вҖӢ

Purpose: Test scenarios before real-world deployment

  • рҹҸ—пёҸ Drag-and-drop infrastructure (roads, hospitals, schools, etc.)
  • рҹ”„ Run agent-based simulation (5вҖ“10 years)
  • рҹ“Ҳ Predict impacts: economic, environmental, social, traffic
  • рҹ“Ҡ A/B testing scenario comparison
  • рҹ“„ Generate detailed reports with Impact Score & Radar Chart

Technologies: Python, Mesa (ABM), FastAPI, Ray
API Port: 8003
Documentation: AnalyticsService/Readme.md

# Create scenario
curl -X POST http://localhost:8003/api/scenarios \
  -H "Content-Type: application/json" \
  -d '{"name": "New Hospital in District 4"}'

# Run simulation
curl -X POST http://localhost:8003/api/scenarios/scenario_123/simulate

4. Emergency Response & Notification Service рҹҡЁвҖӢ

Purpose: Emergency alerts & evacuation guidance

  • рҹҡ‘ Fastest Route For Emergency: Coordinate routes for ambulances

  • рҹ“ұ Multi-channel Alerts: Push, SMS, Email, Web
  • рҹ“Ҡ Cascade Broadcasting: City-wide alert broadcasting
  • рҹ“Қ Geolocation-based: Only alert citizens in affected zones

Technologies: Node.js, Express, MongoDB, Firebase FCM, Kafka
API Port: 3005
Documentation: NotificationService/Readme.md

# Broadcast flooding alert
curl -X POST http://localhost:3005/api/emergency/broadcast-alert \
  -H "Content-Type: application/json" \
  -d '{
    "type": "flooding",
    "severity": "critical",
    "affectedZones": ["zone_1", "zone_2"],
    "message": "Flood Warning"
  }'

# Find emergency route
curl -X POST http://localhost:3005/api/emergency/fastest-route \
  -H "Content-Type: application/json" \
  -d '{
    "vehicleType": "ambulance",
    "fromLocation": {"lat": 16.04, "lon": 108.21},
    "toLocation": {"lat": 16.08, "lon": 108.20}
  }'
